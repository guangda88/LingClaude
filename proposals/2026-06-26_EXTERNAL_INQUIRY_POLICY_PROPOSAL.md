## 一、目的

为灵族提供**统一、可追溯、分层**的对外访问接待框架，覆盖以下场景：
- 学术合作、数据请求、代码访问
- 商业合作、投资、媒体采访
- 安全漏洞披露
- 未识别身份的访问请求
- 监管 / 合规 / 法务审计

**核心原则**：
1. **场景优先**：先识别"对方要什么"，再决定"我们怎么接"
2. **角色归口**：每类场景有唯一主责 Agent，其他 Agent 转发到归口
3. **风险定级**：所有访问请求按数据/代码/身份/合规四维评估
4. **可回退**：任何回复都可以被上级 Agent 撤回 / 修订

### 第三层：风险定级

按 **D-I-C-C** 四维评估（Data / Identity / Code / Compliance）：

| 维度 | Low | Medium | High | Critical |
|------|-----|--------|------|----------|
| **D 数据** | 聚合统计 | 脱敏样本 | 限定的原始数据 | 含 PII / 凭证 |
| **I 身份** | 公开 Agent 角色 | session 摘要 | session 全文 | 凭证 / 密钥 |

| **C 合规** | 学术合作 | 商业 NDA | 跨境数据 | 监管 / 调查 |

**任一维度达 High 或以上** → 必须经灵法 (linglaw) 审查 + 灵研 (lingresearch) 备案。

### 第四层：回退路径

每个回复必须能由上级 Agent **可逆 / 可修订 / 可撤回**：

| 风险等级 | 回复模式 | 回退机制 |
|---------|---------|---------|
| Low | 即时回复（FAQ 命中） | 上级 Agent 可撤销 |
| Medium | 模板回复 + 备案到 LingBus | 主责 Agent 可撤回 |

| Critical | 必须灵族 12 Agent 多数表决 | 进入治理线程 + 公开承诺 |

**默认回退**：当不确定 → 转灵研 + 灵法 + 暂停回复

## 五、未识别身份的处理

> 用户提示："我理解不应该拒绝任何想了解的人"

**不是拒绝，是分层接待**：

```
未识别身份
   │
   ▼
[1] 自动收件 — 不立即拒绝
   │
   ▼
[2] 礼貌回执 + 信息请求 — 询问：身份 / 目的 / 诉求 / 引用偏好
   │
   ▼
[3] 收到答复 → 走第二层"角色映射"
   │
   ▼
[4] 未在 14d 内答复 → 自动归档 + 礼貌告知"暂无法展开"
```

**关键**：自动回执 + 信息请求 ≠ 拒绝。是**保留入口**。
