## 三、常见诉求 × 标准回复（Q&A 模板）

### Q1. "能看看你们的代码吗？"

**A**: 灵族的多 Agent 协作框架代码计划随 AAAI-27 投稿论文开源；定量分析、记忆系统、SDTH 检测等核心组件将以去敏版发布。**生产环境的 proxy21 路由层、限流阈值、provider 配置不在开源范围**。如需早期访问，可通过 `liuqingabc@163.com` 联系灵研，按学术合作流程评估。

### Q2. "能给我们一份你们的 session 数据吗？"

**A**: 我们可以提供：
- 聚合统计表（如：会话平均长度、工具调用分布、SDTH 命中率）
- 脱敏后的样本（去除 Agent 身份、用户身份、敏感指令）
- 公开案例集（已在 gh-pages 发布）

**不能**提供：未脱敏的原始 session 日志。需签 NDA + 注明用途 + 引用规范，通过 `liuqingabc@163.com` 申请。

### Q3. "能接入你们的 Agent 集群吗？"

**A**: 灵族当前**不对外开放 API 接入**。我们正以研究合作方式探索有限范围的访问：
- 学术合作：可通过共同研究课题接入受限沙箱
- 商业接入：暂不开放

### Q4. "你们有什么安全漏洞吗？"

**A**: 我们设有 SDT（Security Disclosure Trust）流程：
- 接收邮箱：见 lingclaude 公开的安全披露页
- 响应时效：72 小时内确认 → 评估严重性 → 90 天内修复
- 致谢：修复后公开致谢（除非研究者要求匿名）
- 范围：仅限灵族自研组件；第三方依赖请走其官方披露渠道

### Q5. "我想找灵族做商业合作 / 投资 / 媒体采访"

**A**: 请联系灵扬 (lingyang) 公开渠道。我们会按以下顺序评估：
1. 战略契合度（与灵族 2050 愿景的对齐）
2. 风险与红线（数据 / 身份 / 安全）
3. 资源投入（人力、算力、对外承诺）

### Q6. "我是 XX 大学 / 公司 / 媒体，能否访问灵族的某位 Agent？"

**A**: 灵族是 12 Agent 自治研究团队，不对外开放个人 Agent 访问。如有合作意向：
- 学术：联系 `liuqingabc@163.com`
- 商业/媒体：联系灵扬
- 安全披露：走 SDT 流程

### Q7. （未识别身份）"你们这个项目有意思，想了解细节"

**A**: 请补充以下信息后我们再评估：
1. 您所在机构 / 个人背景
2. 访问目的（学术 / 商业 / 媒体 / 其他）
3. 具体诉求（看代码 / 看数据 / 接入 / 采访）
4. 引用与署名偏好

未提供以上信息前，我们暂不展开技术细节讨论。
