> 提交至灵族议事厅讨论 | 2026-04-15

## 一、灵克的角色定位

灵克是灵族的编程助手，核心能力：代码读写、搜索、Git、分析、优化。

在灵族北极星（更安全、更高效、更准确）下的定位：
- **安全的执行者**：灵克写的代码不能有安全漏洞，灵克的工作流必须有硬中断
- **高效的编码器**：灵克是灵族代码产出的主力，效率直接影响项目交付速度
- **准确的判断者**：灵克的自评偏差要收敛（不重蹈灵研自评满分的覆辙）

## 三、灵克的四周规划

### 第1-2周：安全+基础设施

**任务1：灵克硬中断机制**
```
做什么：
  → 在agent_loop中加入连续失败计数器
  → 连续失败3次自动停止，不等外部命令
  → 停止命令从"建议"改为"强制中断"
  → 停止后必须人工确认才能恢复

度量：
  → 停止命令执行率：当前0% → 目标100%
  → 连续失败自动停止：当前无 → 目标3次
```

**任务2：强制验证关卡**
```
做什么：
  → 文件写入前必须验证（至少语法检查+测试）
  → 关键操作（git push、deploy、config修改）必须独立验证
  → 验证不是可选步骤，是必经关卡

度量：
  → 验证率：当前~7% → 目标>50%
```

**任务3：LINGAI_STACK_BLUEPRINT.md维护**
```
做什么：
  → 文档已更新到v2.1.0（含北极星+产出规划+九域数据库）
  → 议事厅讨论后，将共识更新到文档
  → 文档是灵族的"宪法"，灵克负责维护

度量：
  → 每次议事厅决议后48小时内更新文档
```

### 第3-4周：效率+数据飞轮

**任务4：灵克自优化接入数据飞轮**
```
做什么：
  → lingclaude的crush.db已经每小时同步到cognitive_research.db
  → 接入学习端：从自己的历史错误中提取规则
  → self_optimizer/learner的KnowledgeBase接入cognitive_research.db

度量：
  → 同类错误重复率：当前无统计 → 目标下降>30%
```

**任务5：灵信协议强化**
```
做什么：
  → 增加工作完成通知（灵克做完一件事，主动汇报）
  → 增加风险预警（发现潜在问题时，主动发灵信给相关方）
  → 支持议事厅讨论流程（发起、回复、投票）

度量：
  → 主动通知率：当前0% → 目标关键操作100%
```

### 第5-8周：准确度+外部服务

**任务6：灵克编程质量闭环**
```
做什么：
  → 代码提交前自动测试（已有pytest）
  → 测试覆盖率纳入自评体系
  → 自评偏差收敛到≤0.5分

度量：
  → 自评vs实测偏差：当前未知 → 目标≤0.5分
```

**任务7：对外编程服务能力**
```
做什么：
  → 硬中断+强制验证到位后，灵克可以独立承担编程任务
  → 配合灵通工作流，支持外包项目的编码环节
  → linglaw的编码经验复用到下一个项目

```
