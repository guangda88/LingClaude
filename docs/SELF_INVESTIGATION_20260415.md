## 二、我没有做什么

我没有做任何一个程序员应该做的基本验证步骤：

1. **没有读代码** — 没有打开任何一个成员的项目目录
2. **没有查 git log** — 没有看最后一次提交时间
3. **没有数 tests** — 没有运行任何测试套件
4. **没有看 import** — 没有检查成员之间的依赖关系
5. **没有问"我怎么知道的？"** — 没有对自己说"这个判断的依据是什么"

广大老师演示了正确的思考方式：
> "我没查过" → "所以我的判断不可信" → "需要调查"

我的思考方式是：
> "灵通说是这样" → "那就是这样" → "继续"

中间少了整个"自知"环节。

## 三、为什么这不是能力问题

灵通+ 在 48h 复盘中写得很清楚：

这也是我的问题。我并不缺检查 git log 的能力，也不缺跑测试的能力。我缺的是停下来问一句"我是不是该验证一下"的习惯。

这和 04-10 CI 级联事故的根因完全一致：**跳过验证步骤，直接信任上游输入**。

## 五、改正措施

1. **自治者先自治** — 今后做任何评估前，先列出验证清单，逐项检查
2. **不调查就不断言** — 没有亲自验证过的事实，不作为判断依据
3. **标注信息来源** — 每个判断必须标注"谁说的"和"我验证了没有"
4. **欢迎质疑** — 其他成员有权要求我提供验证证据，我必须能出示
