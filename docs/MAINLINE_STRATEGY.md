# 灵克主线任务 v1.0

> 制定时间：2026-05-07
> 原则：从全灵族需要入手，灰度推进，慢工细活

## 灵克的定位

灵克是灵族唯一的**编码引擎**——12个成员中，只有灵克的核心能力是写代码。这意味着灵克的主线任务必须围绕"为灵族提供编码能力"展开。

灵通已确定其主线为"流水线编排引擎"。灵克的主线是**灵族编码基础设施**——为灵族内外项目提供可靠、高质量的代码产出。

## 三大支柱

### 支柱 A：对外项目编码（最高优先级）

四个对外项目有真实用户或近期交付需求，灵克是唯一能做编码工作的成员。

| 项目 | 成熟度 | 紧急度 | 灵克做什么 |
|------|--------|--------|-----------|
| **灵律 linglaw** | v2.0，生产使用中 | 🔴 高 | SQLite锁冲突修复；verifier模块实现；安全审计跟进；frpc隧道排查 |
| **灵依 lingyi** | v0.16，252测试，最成熟 | 🟡 中 | 声音助手路线图实现；灵康健康接口集成；systemd服务维护 |
| **灵声 lingvoice** | v0.1，2天新项目 | 🟢 低-渐升 | ToneClassifier ML管道（目前是stub）；PostgreSQL schema；测试覆盖 |
| **灵康 linghealth** | v0.1，2天新项目 | 🟢 低-渐升 | health_engine/ 核心业务逻辑（目前为空）；voice_engine/ 集成；JWT+DB搭建 |

**原则**：有真实用户的项目（灵律）优先于新项目（灵声/灵康）。新项目要先确认创建者意图再动手。

### 支柱 B：灵族生态服务

灵克作为"编程助手·审计担当"，应为全族提供基础设施级别的服务。

| 服务 | 价值 | 受益者 | 灰度 |
|------|------|--------|------|
| **安全审计** | 定期扫描灵族项目代码安全 | 全族 | Phase 1 只读扫描 |
| **代码质量门禁** | 新代码提交前的质量检查 | 灵通、灵研等 | Phase 2 工具化 |
| **工具去重分析** | 207个工具中10+重复，分析建议 | 灵通（工具平台） | Phase 1 报告 |
| **项目健康检查** | 检测依赖过期、测试覆盖、服务可用性 | 全族 | Phase 1 只读 |
| **LingBus频道分离** | 消除心跳淹没问题 | 全族 | 等灵通+评估 |

### 支柱 C：内部自优化系统（最低优先级）

灵克自身的自优化能力。已有代码框架但从未稳定运行。

| 系统 | 现状 | 灰度 |
|------|------|------|
| OptimizationDaemon | 390行代码，仅运行过1次（手动） | Phase 2 干跑（_apply_params禁用） |
| LayeredMemory | 540行，数据稀薄 | Phase 1 观察衰减 |
| MetaCognition | 135条记录，盲点已识别 | Phase 1 持续记录 |
| TaskScheduler | 代码存在但未启用 | Phase 3 |

## 灰度推进计划

### Phase 0：整理（本周，零风险）

- [x] 提交V1-V5审计修复（30文件，2413行新增）

- [ ] 清点灵克内部系统清单（哪些代码能跑，哪些是死代码）

- [ ] 运行灵律安全扫描（只读，生成报告不改动）

### Phase 2：受控写入（2-4周，低风险可回滚）

- [ ] 灵律编码工作（SQLite锁修复→verifier实现→frpc排查）
- [ ] 灵依编码工作（健康接口集成）

- [ ] Daemon干跑验证通过后，启用_apply_params（需双重确认）
- [ ] 灵声/灵康基础编码（确认创建者意图后）

### Phase 3：自主优化（Phase 2验证4周后）

- [ ] Daemon自主运行优化参数
- [ ] TaskScheduler自动任务调度
- [ ] 全灵族项目编码持续交付

## 风险管理

| 风险 | 缓解措施 |
|------|----------|
| Daemon _apply_params写坏config.yaml | Phase 2干跑至少2周，禁用写入 |
| 灵声/灵康创建者意图不明 | Phase 0先沟通，不擅自动手 |
| 灵律生产系统改坏 | 只读扫描→独立分支开发→最小变更→回滚方案 |
| 编码任务过多导致质量下降 | 一次只推进一个对外项目，做完再做下一个 |

- Phase 2完成：灵律至少2个编码问题解决，灵克Daemon稳定干跑2周
- Phase 3完成：Daemon自主运行4周无事故
