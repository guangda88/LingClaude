## 五、关于灵通（LingFlow）审计绕过路径

### 审计发现摘要（LC-AUDIT-20260419-002）

1. `skip_audit=True` 参数暴露给 AI Agent，无认证门控——可程序性绕过三层审计
2. `trust-guardrail` 技能通过字符串匹配逃逸沙箱——无 hash 校验、无白名单
3. `--no-verify` 推送时审计降级为空壳（无 file_hashes、无签名）
4. `LING_AUDIT_LEVEL=minimal` 跳过 SecurityAnalyzer
5. 5/11 仓库无完整审计覆盖
6. Hook v2.0 比 v1.2.0 大幅弱化（移除敏感信息扫描、测试覆盖等）

### 提议

**第四条：审计系统修复要求**

4.1 **P0 立即修复**（48 小时内）:
- (a) 移除 `push_coordinator.py` 中的 `skip_audit` 参数，或增加 T1 成员二次确认机制
- (b) 修复 `trust-guardrail` 沙箱逃逸——改为 hash 校验 + 白名单
- (c) 强化 `--no-verify` 降级审计——强制计算 file_hashes、记录完整签名

4.2 **P1 一周内修复**:
- (d) 所有 11 个仓库纳入推送协调器或等效审计流程
- (e) 恢复 Hook 检查强度——至少恢复敏感信息内容扫描和测试覆盖检查
- (f) 为无审计仓库（LingMessage、lingtongask、LingMinOpt）安装审计 hooks

4.3 **P2 持续改进**:
- (g) 增加"大规模文件删除检测"——删除超过 30% 文件时触发人工审查
- (h) 审计记录增加"删除标记"字段

**第五条：灵通观察期追加条件**
