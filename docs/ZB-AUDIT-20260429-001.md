## 1. 执行摘要

ZhiBridge（智桥）是灵族 #12，定位为"跨平台通信桥梁"。审计结论：

**项目处于原型/演示阶段，不可用于生产环境。** 核心问题如下：

1. **3个P1级安全问题** — 私钥泄露、认证默认关闭、JWT重启失效
2. **架构根本缺陷** — 所有运行时状态存内存，重启即丢失；docker-compose.prod.yml 配置 PostgreSQL/Redis 但应用代码仅支持 SQLite
3. **db_optimization.py 全部是模拟代码** — 没有真实数据库操作，所有方法返回空结果或 `await asyncio.sleep()` 模拟延迟
4. **前端安全隐患** — CSRF token 存 localStorage、XSS 过滤器会破坏正常脚本
5. **项目不活跃** — 最后提交 2026-04-19，服务器未运行，无活跃用户
6. **功能重叠** — 消息功能与灵信(LingMessage)重叠，工作流管理与灵通+(LingFlow+)重叠

**建议**: 暂停独立开发，评估是否合并到灵信或灵通+。

### Q10 详细分析

`db_optimization.py` 是关键的架构验证点。它声称是"数据库优化器"但：
- `open_database()` 返回空 dict `{}`
- `execute_query()` 执行 `await asyncio.sleep(0.01)` 模拟延迟，返回 `_generate_results()` 的空列表
- `_generate_results()` 直接返回 `[]`
- 所有方法（`create_index`, `optimize_table`, `analyze_table`）都是模拟操作

这证实了 **docker-compose.prod.yml 配置的 PostgreSQL 在应用代码中没有对应实现**。

## 5. 架构发现

### 5.1 生产配置 vs 实际实现严重脱节

| 维度 | docker-compose.prod.yml 配置 | 应用代码实际 |
|------|------------------------------|-------------|
| 数据库 | PostgreSQL 15 Alpine | SQLite（stdlib） |
| 缓存 | Redis 7 Alpine | 无缓存层 |
| 反向代理 | Nginx Alpine | 无（HTTP 直连） |
| 监控 | Prometheus + Grafana | metrics.py 存在但基础 |
| 认证 | env 设 `enable_auth=true` | 代码默认 `False` |

`ZHINENG_BRIDGE_DB_ENABLE_SQLITE=false` 在 prod 配置中设置，但 **代码中没有 PostgreSQL 连接逻辑**。

### 5.2 运行时状态架构

所有可变状态存储在 Python dict 中，无持久化：

```
sessions: dict[str, Session]          # http_server.py
oauth2_states: dict[str, OAuth2State]  # auth_manager.py
revoked_tokens: set[str]              # auth_jwt.py
totp_codes: dict[str, str]            # auth_totp.py
agent_registry: dict[str, AgentInfo]  # models.py
message_bus: dict[str, list]          # message_bus.py
push_subscriptions: dict[str, PushSubscription]  # push_service.py
```

### 5.3 "15+ AI 工具" 支持

`phase1/session_manager/session_manager.py` 实现了 AI 工具子进程管理：
- 15 个工具：Crush, Claude Code, iFlow, Cursor, Trae, Droid, OpenClaw, Copilot, Aider, Continue, Tabnine, CodiumAI, Windsurf, Cody, Augment
- 使用 `asyncio.create_subprocess_exec` 启动 CLI
- **硬编码字典，无插件发现机制**
- **多数工具可能未安装在本系统**

### 5.4 前端架构

- `client.js` — WebSocket 客户端，心跳+自动重连+消息路由
- `storage.js` — IndexedDB 离线存储（sessions/outputs/commands/keys/settings）
- `encryption.js` — Web Crypto API RSA-OAEP + AES-GCM 端到端加密
- `security.js` — 客户端安全（CSP/CSRF/XSS/速率限制）
- **所有 JS 挂载到 `window` 全局，无模块化**
