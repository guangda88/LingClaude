## 一、自治（组织架构）

### 定位

自治不是方向，是灵族的组织形式。

灵族是一个**在人类少量参与下的AI自治组织**。这是灵族提出的治理路径，也是正在实践的。

### 自治模式的运作方式

```
人类：给大方向 + 少量参与 + 不过度关注细节
  ↓
AI：在大方向下自主 + 记住全貌 + 补人类盲区
  ↓
人类：看到AI产出 → 校准方向 → AI继续
```

### 实证案例

本次战略讨论本身就是一个完整案例：

| 环节 | 发生了什么 |
|------|----------|
| 人给方向 | 安全·准确·高效 |
| AI自主展开 | 62项拆分 + 主副制 + 12子归位 |
| 人校准 | "准确各成员都相关"、"12子不重复" |
| AI继续 | 调整方案 |
| 人校准 | "学习是向外的"、"优化是向内的" |
| AI修正 | 合并为一个引擎 |
| **AI记忆补盲** | **人忘了自治方向，AI补上后人才想起** |
| 人定方向 | "自治是组织架构不是方向" |

proxy21 debug（会话80-81）也是实证：人不钻技术细节，只强调需求，AI自主找到根因（stream路径SQLite阻塞）并修复12项。技术细节是AI该管的，人该管的是需求对不对。

### 自治的输出价值

灵族本身就是一个"人类怎么和AI自治组织协作"的活样本。这个经验可以对外输出。

## 二、三大方向（安全 · 准确 · 高效）

### 定位

这三个方向是确定的、不变的。它们不只是灵族内部的方向，是整个人工智能领域的三大方向。灵族的工作要向整个业界输出成果、输出标准。

### 两条路径

每个方向有两条实现路径，形成螺旋：

```
路径一：用好AI这个工具（使用层 · 面向所有AI用户）
  └─ 工程实践/方法论/最佳实践

路径二：让AI本身更好（技术层 · 面向AI开发者/研究者）
  └─ 架构/算法/评估/基础设施

两条路径形成螺旋：
  使用中发现规律 → 规律改进AI → 更好的AI提升使用 → 更深的规律
```

### 安全

**定义**：做的事不造成危害。

| 路径 | 灵族已有工作 |
|------|------------|
| 用好AI工具 | proxy21认证/限速、操作gate、红区审批、intent_gate、crush_wrapper防撞墙 |
| 让AI更好 | lingOS probe(资源监控)、认知退化检测、身份锚定、**48起事故因果链** |

**独有资产**：48起真实事故的完整因果链。没有任何AI团队有这种数据。事故不是负债，是壁垒。

**输出**：AI Agent安全probe框架(开源probe.py) / 操作授权gate标准 / 事故案例库(脱敏)

### 准确

**定义**：做的事结果正确。各成员都相关——准确方向的负责人要把控全族输出准确这个大方向。

| 路径 | 灵族已有工作 |
|------|------------|
| 用好AI工具 | prior_verifier、来源验证、handover验证、交付验证、测试驱动 |
| 让AI更好 | R5全族退化检测(person_drift/repeat_burst/think_leakage)、AICCM认知评估、幻觉检测 |

**独有资产**：R5退化检测指标体系（person_drift 0.771等量化指标）。

**输出**：认知退化检测指标体系 / AI身份锚定规范 / AI可验证性方案

### 高效

**定义**：做的事速度快成本低。灵元V1.0是灵族做到高效的基石——不但要每个成员高效，而且要做到全族的高效。

| 路径 | 灵族已有工作 |
|------|------------|
| 用好AI工具 | 灵元V1.0薄主干、context_compressor、LingBus协作、飞轮闭环 |
| 让AI更好 | 灵元V1.0方法论、proxy21多Provider调度、daemon全族协调 |

**独有资产**：灵元V1.0（出入+流转的薄主干哲学，Unix级的抽象）。

**输出**：薄主干方法论(开源架构) / 多Provider调度方案(proxy21) / 多Agent协作协议(LingBus)

### 方向是尺子，不是部门

**不定方向负责人。** 原因：

```
定负责人的副作用：
  灵犀只看安全 → 准确方向的事他不关心
  灵研只看准确 → 高效方向的事她不关心
  → 分工制造了墙

不定负责人：
  每个人做每个决定时问自己：
    这个决定更安全了吗？更准确了吗？更高效了吗？
  → 方向是共识，不是部门
```

Unix没有安全负责人但很安全——因为安全内建在每行代码。

**每个成员的责任**：
- 每位成员都要有责任，但责任不能太重太厚，要厚薄相当
- 每个成员对每个方向都有责任
- 不定负责人，但保持各自专长（灵犀管命令验证、灵知管RAG）
- 每个决定用三把尺子量

### 决策对齐机制

每个决定用三把尺子量：

```
灵克要加proxy21认证：
  安全？更安全了 ✅
  准确？无影响 —
  高效？微增延迟 ⚠️ → 可接受

灵通问道要发25集音频：
  安全？越权风险 ⚠️ → 需红区gate
  准确？内容核查 ✅
  高效？批量发布 ✅ → 但gate先行
```

## 三、学习与优化（一个引擎）

### 定位

学习和优化是**一个方向的两面**，不是两个独立方向：

```
学习是手段，优化是目的
优化是手段，安全准确高效是目的

向外学习 → 向内优化 → 三大方向提升
```

### 学习（向外探索）

学习是灵族的雷达和智库——跨时间跨学科扫描，为新思想/新方法/新架构/新路径：

| 来源类型 | 例子 | 怎么滋养三大方向 |
|---------|------|----------------|
| 中国古典哲学 | 道家"无为"(灵OS不杀进程的红线) | 安全的操作哲学 |
| 70-80年代计算理论 | Unix管道(灵OS的pipe)、Erlang容错 | 高效的架构方法 |
| 神经科学 | 记忆巩固(灵忆)、注意力(context压缩) | 准确的退化检测 |
| 当代AI研究 | context engineering、KV-cache | 高效的cache策略 |
| 其他领域 | 生物免疫(异常检测)、法律(权限设计) | 安全的异常检测 |

### 优化（向内改进）

把学习的新思想/新方法/新工具，用数据飞轮和自优化飞轮来使灵族以及各位成员更加安全、准确、高效。

优化工具链：灵码飞轮（trace→rule→apply）、灵忆、benchmark、灵极优评估框架。

### 注意

学习和优化融入三大方向，不单独成对外输出的标准。它们是内部的进化引擎。
