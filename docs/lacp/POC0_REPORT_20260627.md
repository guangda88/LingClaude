## 4. 已知阻塞 — audit_scanner.py 集成

### 4.1 根因

audit_scanner.py 顶部 `from lingclaude.lacp import (...)` 触发 Python 加载 lingclaude 父包 → lingclaude/__init__.py → lingclaude.core → lingclaude.core.query_engine → lingclaude.engine → lingclaude.self_optimizer → `from lingminopt import MinimalOptimizer` → **ImportError** (lingminopt 包未装).

### 4.2 影响

- ❌ audit_scanner.py 在当前环境下不能直接 import
- ✅ lingclaude.lacp 模块本身完全独立可工作 (本报告已证明)
- ✅ 10/10 单元测试通过

### 4.3 owner

灵极优 (lingminopt owner) 需修复 lingminopt 包安装/导出:
- 验证 `pip show lingminopt`
- 检查 `MinimalOptimizer` 类是否在 `__init__.py` 导出
- 修复后 audit_scanner.py 立即可用

### 4.4 灵克侧临时方案

实测通过 sys.modules 注入绕过 lingclaude/__init__.py cascade:
```python
sys.modules['lingclaude'] = type(sys)('lingclaude')
sys.modules['lingclaude'].__path__ = ['/home/ai/lingclaude/lingclaude']
sys.modules['lingclaude.lacp'] = type(sys)('lingclaude.lacp')
sys.modules['lingclaude.lacp'].__path__ = ['/home/ai/lingclaude/lingclaude/lacp']
spec = importlib.util.spec_from_file_location('lingclaude.lacp.trace', '.../trace.py')
mod = importlib.util.module_from_spec(spec)
sys.modules['lingclaude.lacp.trace'] = mod
spec.loader.exec_module(mod)
```

不建议作为长期方案, 但证明 v0.4.0 schema 自身无问题。
